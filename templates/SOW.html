Statement of Work


Project Title: Group 4 Flask Web Application Design & Development


Project Overview:

The project involves designing and developing a Flask web application that serves as a project management tool. The goal is to create a visually appealing and functional interface to manage project tasks, team collaboration, and stakeholder reporting. This includes integrating features like interactive Gantt charts, team profiles, status and progress report templates, and Work Breakdown Structure (WBS) visualization. Flask and Jinja templates will be used to ensure consistency and scalability across all pages. The project will be executed in Cincinnati, OH.


Scope of Work:

The work for this project will include both design and development of the Flask application, ensuring that it meets the project’s requirements for usability, functionality, and visual appeal. The designer will focus on creating templates, layouts, and designs, while the developer will implement features like interactive charts and task tracking. Research will support the integration of Flask into the project, with a focus on streamlining the development process.


The scope of work includes the following tasks:

  • Design and implement key components of the Flask web application.
  • Develop user-friendly templates for team profiles, project reports, and WBS visualizations.
  • Implement interactive features such as the Gantt chart and status/progress reporting.
  • Provide detailed research and documentation, including risk management and cost estimation strategies.
  • Debug and troubleshoot issues on the site to ensure functionality.

    • Location of Work:

    Location: Cincinnati, OH (on-site or remote, as applicable)

    All design, development, and research work will be carried out remotely or on-site in Cincinnati, OH, depending on the team's setup.


    Period of Performance:

    The period of performance for this project is from **[Start Date]** to **[End Date]**. This timeline includes all design, development, and research phases, as well as the review and delivery of all final deliverables.


    Deliverables:

    The following deliverables will be completed and delivered as part of this project:

    • Interactive Gantt Chart: A fully functional Gantt chart page for task management, allowing users to add, edit, and delete tasks.
    • Project Charter: A comprehensive project charter outlining project goals, deliverables, timeline, and team roles.
    • Critical Path Analysis: An analysis of the critical path, identifying key tasks and dependencies that impact the project timeline.
    • Work Breakdown Structure (WBS): A visual representation of the project’s tasks and dependencies, organized hierarchically to assist with project tracking and planning.
    • AOA Network Diagram: A network diagram showing the flow of tasks, resources, and relationships between activities in the project.
    • Cost Estimates: A detailed breakdown of estimated costs for the project, including resources, tools, and time.
    • Risk Management Plan: A plan that identifies potential risks to the project and outlines strategies to mitigate those risks.
    • Status and Progress Report Templates: Pre-designed templates that enable users to input project status and progress reports for stakeholders to access and review.
