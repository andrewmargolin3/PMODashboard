Overview

Critical Path Analysis (CPA) is a project management technique used to determine the sequence of essential tasks that must be completed on time for the project to meet its deadline. The "critical path" represents the longest path through the project timeline, ensuring that all tasks are completed on schedule. Time to completion is often notated by a number, such as (4) being 4 days to complete. When the tasks are added up by dates, that is your critical path and the shortest time of completion.

Importance of Critical Path

Identifying the critical path allows project managers to focus on tasks that have the highest impact on the project’s completion date. Delays in tasks on the critical path could result in the entire project being delayed, so these tasks must be monitored closely.

Critical Path

A. Project Planning and Group Roles (2)
D. Create GitHub Repository (1)
E. Creation of Main Pages (21)
F. Styling and Cleanup (7)
G. Monitoring and Testing (7)

This is our critical path and will elapse 38 days. There are additional paths that can be taken, but this is the shortest path hitting every main task.
